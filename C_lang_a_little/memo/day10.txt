[Call-by-value, Call-by-reference]

[Call-by-value]
함수를 정의할 때 매개변수를 기본 형태의 변수로 지정하면 함수를 호출할때 단순히 값을 전달합니다.

이 방식은 값을 복사하여 전달하므로 해당 값은 함수 지역 내에서만 사용되고 이후에는 자연스럽게 소멸합니다.

[Call-by-reference]
코드에서 실제 값의 변경이 이루어지게 하려면 함수에 인자를 전달할때 실제 값을 전달해야 합니다.
그 실제 값은 메모리 공간에 저장되어 있습니다. 따라서 값이 저장되어있는 메모리 공간의 주소값을
전달해주면 실제 값을 변경할수 있습니다.

[구조체]
구조체는 하나이상의 변수를 한꺼번에 묶어서 만드는 ' 새로운 유형의 자료형' 이라고 부릅니다.

사용자에 의해 정의되므로 '사용자 정의 자료형' 이라고 부르기도 합니다.

배열과 구조체의 차이점
배열 : 고정크기, 같은 자료형
구조체 : 동적할당, 다양한 자료형

필드
데이터를 분류하는 속성 -> 이름, 나이, 성별

레코드
속성으로 분류되는 값 -> 홍길동, 20, 남

1)구조체 정의

struct 구조체명{ //초기화 불가능
	자료형 변수명1; -> 필드
	자료형 변수명2;
	자료형 변수명3;
}

2)구조체 변수 생성
struct 구조체명 구조체변수명;

3)구조체 필드 정의하면서 구조체 변수를 미리 생성하는법

struct 구조체명{ //초기화 불가능
	자료형 변수명1; -> 필드
	자료형 변수명2;
	자료형 변수명3;
} 구조체 변수명1, 구조체 변수명2, ......;

생성된 구조체 변수 내 필드에 값 저장하기.

구조체변수.필드명 = 값;

문자열의 경우 strcpy()함수를 사용해야 한다.

"홍길동" -> 문자로 된 집합

strcpy(구조체변수.필드명, "값");
